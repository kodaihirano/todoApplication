BattleUnitクラス¶

  • enchant.Sprite ← Unit ← BattleUnit という継承関係。
  • Unitクラスでは、スプライトに関する初期設定を行う。
  • BattleUnitクラスではステータスに関する初期設定を行う。
  • 引数に次のようなJSON(もどき)を受け取り、インスタンスを生成する。
  • プロパティは今後増えるかも。
{
  "spriteConf": {
    "sizeX": 32,
    "sizeY": 32,
    "frame": 0,
    "scaleX": 1,
    "scaleY": 1,
    "imagePath": "/public/images/sampleRPG/chara0.gif",
    "sx": 192,
    "sy": 0,
    "sw": 96,
    "sh": 128,
    "dx": 0,
    "dy": 0,
    "dw": 96,
    "dh": 128
  },
  "name":   "unitA",
  "hp":     30,
  "hpMax":  30,
  "en":     15,
  "enMax":  15,
  "atk":    10,
  "def":    5,
  "spd":    5,
  "actionSelection": function() {...},
  "skills": {"attack": "スキルのJSONへのパス", ...}
}
  • 画像やJSONはpreload()しておく。
  • actionSelectionには、ガンビットで設定した行動決定に関する情報を格納する。
  • ガンビットでどの程度のことができるかわからないので、担当と相談したい。
  • skillsには習得しているスキルの名前を文字列で格納する。
  • スキルオブジェクトは事前にグローバルに宣言しておき、インスタンス生成時にeval()を用いてスキルへの参照を得る。

    // 3対3のバトル(敵キャラクター数は固定)において、

    }

    }

  • 敵キャラクター数について固定しているのは、BattleUnitを配列で管理している都合上。
    • 仲間が増えたら、[allyUnits[1]]ブロック、[allyUnits[2]]ブロックを解禁すれば良い。
  • コンパイルについては対応するJavaScriptのソースコードに変換する。文字列から関数をパースできるのでこれでOK。
  • 上記のCのコードは次のように変換される。
    • 変数名は基本キャメルケースで書く予定。
    • 第一引数はself(BattleUnit)、第二引数は効果の対象(BattleUnit、全体に効果のあるスキルの場合はBattleUnitの配列)
    • 一番左のユニットなら、この関数をglobal変数gameData.party[0].actionSelectionにセットする。
function(self, allyUnits, enemyUnits) {

RPG設定について¶

  • ENについては、ポケモンのPPと言うよりは、個人的にはシャドバで言うコストのようなものにするのがいいのではないかと思っています。
    • どのユニットも初期値2、毎ターン2回復、ためるコマンドで1回復、Max10
    • 通常攻撃で1消費、スキルでたくさん消費、足りなければ失敗。
    • 長期戦になりやすいので何かしらのテコ入れが必要。
  • スキルの中身については色々考えなければならないです。
    • 皆さんのアイデア募集します。コメント下に書いといてください。

BattleSceneへの遷移¶

マップ等からバトルへ遷移する方法。

// '/public/json/battle/battleX.jsonをpreload()しておく。
let json = game.assets[JSON_BATTLE_BATTLEx];
game.pushScene(new BattleScene(json));

因みにbattleX.jsonの中身はこんな感じに書かれている。

{
  "battleName": "battle1",
  "num":  3,
  "enemyNames": ["unitB-1", "unitB-2", "unitB-3"],
  "enemyFilePaths": [
    "敵データへのパス",
    "敵データへのパス",
    "敵データへのパス"
  ]
}

スキル強化について¶

  • Skillクラスのコンストラクタにjsonデータを渡すことで具体的なスキルのインスタンスを生成し、BattleUnit.skillsに格納しておく。インスタンス変数名はスキル名と同じ。
  • スキル強化については、BattleUnit.skills.スキル名.bonusを書き換える。
    • 現状は、bonus.rate(初期値1.0)のみが存在し、これはスキルの効果値の補正倍率を表す。例えば、通常10ダメージのスキルでbonus.rate = 1.2なら12ダメージとなる。

  • 火炎放射(全体攻撃、固定ダメージ)
  • リペア(単体回復)
  • チャージ(EN増加)

  • 全体攻撃
  • 単体回復
